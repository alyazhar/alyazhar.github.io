Building PICKR —
a platform for civic voice.

PICKR is a civic engagement mobile application that lets people create polls, join consultations, and make collective decisions in a simple, transparent way. Across this semester I used Jetpack Compose, Navigation, Room Database, Firebase, GPS, and API integration to turn that idea into a working Android application — built in service of community participation.

Lab Journal · 01–05

From layout to local storage

Five labs, each a building block of PICKR — from the first sketch of the interface to a fully persistent, multi-screen application.

Lab 01

User Interface with Jetpack Compose

Structuring the first version of PICKR's interface using Column, Row, and Box.

Reflection Through Lab 1, I learned how to build a structured user interface using Column, Row, and Box in Jetpack Compose. I designed the initial version of Pickr by creating a clean layout inspired by Pollie while adapting it to support SDG 16 goals. This lab helped me understand how different layout components work together to create an organized and user-friendly interface.
Lab 02

Interaction & State Management

Making the interface respond instantly using remember and mutableStateOf.

Reflection In Lab 2, I learned how to manage user interactions and dynamic UI updates using remember and mutableStateOf. Users could create a poll question and immediately see the updated content on the screen without refreshing the application. This experience helped me understand the importance of state management in creating responsive and interactive mobile applications.
Lab 03

Material Design 3

Refining PICKR's visual language with a consistent purple theme and expandable cards.

Reflection Lab 3 focused on improving the visual design and user experience of Pickr using Material Design 3. I implemented a consistent purple theme, custom typography, card-based layouts, and expandable animations to make the application more engaging and organized. Through this lab, I learned how thoughtful design choices can improve both usability and accessibility.
Lab 04

Navigation Compose

Connecting Login, Home, Create Poll, Results, and Profile into one app flow.

Reflection In Lab 4, I implemented Navigation Compose to transform Pickr into a multi-screen application. I created navigation flows between the Login, Home, Create Poll, Results, and Profile screens while using a ViewModel to manage shared data across screens. This lab strengthened my understanding of navigation architecture and state management in modern Android development.
Lab 05

Room Database

Giving PICKR persistent memory with Entity, DAO, Repository, and Flow/StateFlow.

Reflection Lab 5 introduced persistent local storage through Room Database integration. I implemented Entity, DAO, Repository, and ViewModel components to store polls permanently and automatically update the interface using Flow and StateFlow. This lab helped me understand how real-world applications manage data persistence and maintain consistency between the database and UI.
